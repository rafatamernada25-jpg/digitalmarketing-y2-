1. Execution Flow: Synchronous vs. Asynchronous

Synchronous Execution (Blocking)

The code runs sequentially. When an HTTP request is made, the program halts and waits for the server to respond before moving to the next line.

Behavior: The UI may freeze or the browser may become unresponsive until the data arrives.

Asynchronous Execution (Non-Blocking)

The code initiates the request and immediately continues executing the next lines without waiting for the server.

Behavior: Uses patterns like "async/await" to handle data later. The UI remains responsive.

2. Data Retrieval and Methods

Once a request is sent, the goal is to retrieve data and make it usable for the application.

The Request Methods (Verbs)

GET: Retrieve data
POST: Send new data
PUT: Update data
DELETE: Remove data

The Payload and Headers

Headers: Meta-data sent with the request (e.g., Content-Type: application/json).

Body: The actual data being sent (used in POST/PUT).

3. Processing the Response

Raw data usually arrives as a string stream. You must parse it into a native data structure.

JSON (JavaScript Object Notation): The industry standard. Text easily converted into objects.

XML: Older, more verbose, tag-based format.

Example (JavaScript/Fetch):

async function fetchData() { const response = await fetch('https://api.example.com/data'); const data = await response.json(); // Parsing step console.log(data); }

4. Error Handling & Summary

Networks are unreliable. A robust application assumes requests will fail eventually and plans for it.

HTTP Status Codes

2xx (Success): Everything worked (e.g., 200 OK).

4xx (Client Error): You made a mistake (e.g., 404 Not Found).

5xx (Server Error): The server crashed (e.g., 500 Internal Error).

5. Final Assessment

Test your technical expertise. Hints and corrections will guide your learning.

1. Analyze this code:
const response = fetch('api/data'); console.log(response);
Why will the console show "Promise {pending}" instead of the data?

Hint: Fetch is asynchronous. Without instructions to wait, JS moves to the next line immediately.

Correction: You are logging the Promise object itself. You must 'await' the fetch to get the actual Response.

2. You send POST data but the server doesn't read it. What is likely missing?

Hint: The server needs to know how to interpret the raw bytes you sent in the body.

Correction: Headers tell the server the format of your data so it can parse it correctly.

3. Why check 'response.ok' before calling '.json()'?

Hint: If a server crashes, the body usually contains HTML error text, not the JSON you expect.

Correction: Parsing an error page as JSON will crash your app. Always verify success first.

4. Which HTTP method is used to retrieve a list of products?

Hint: Think of the verb used for "getting" information without changing it.

Correction: GET is the standard method for retrieving resources from a server.

5. What does an error code starting with 5 (e.g., 503) indicate?

Hint: 4xx is Client-side, 5xx is...

Correction: 5xx codes mean the server failed to process a valid request (crash or maintenance).

6. What is the primary benefit of the Asynchronous model?

Hint: Think about being able to scroll while data is being fetched.

Correction: Async execution is "non-blocking", allowing the UI to stay responsive.

7. Where should sensitive information like Tokens be placed?

Hint: You want to avoid showing credentials in the browser's address bar.

Correction: Headers are the secure, standard way to pass authentication metadata.

8. Which data format is most used by Web APIs today?

Hint: It stands for JavaScript Object Notation.

Correction: JSON is faster and easier to parse than XML in modern environments.

9. In a Try/Catch block, what does the 'Catch' section do?

Hint: It acts as a "fallback" when the 'Try' block fails.

Correction: Catch handles exceptions (like no internet) to prevent the whole app from crashing.

10. What does a server physically send back?

Hint: Everything on the web travels as raw text that needs to be parsed.

Correction: Data travels as a stream of characters. `.json()` turns that text into an object.
